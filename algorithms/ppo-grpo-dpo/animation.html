PPO vs GRPO vs DPO

RLHF 三大算法原理对比

PPO

L = -E[min(
  rt · At,
  clip(rt, 1-ε, 1+ε) · At
)]

rt = πθ(a|s) / πold(a|s)
At = Rt - V(st)
rt: 重要性采样比（新旧策略概率比）
At: Advantage = reward - value baseline
clip: 限制更新幅度，防止策略崩溃
V(s): 需要额外的 value model

DPO

L = -E[log σ(β · Δ)]

Δ = log πθ(yw)/πref(yw)
  - log πθ(yl)/πref(yl)

隐式 reward:
r(y) = β · log πθ(y)/πref(y)
yw: chosen (人类偏好的回答)
yl: rejected (人类不偏好的)
β: 控制偏离 ref 的程度
无需 RM: 直接从偏好对优化

GRPO

L = -1/G · Σ min(
  ri · Ai,
  clip(ri, 1-ε, 1+ε) · Ai
) + β · KL

Ai = (ri - mean) / std
(组内 z-score，无需 Value)
G: 每个 prompt 采样 G 个响应
Ai: 组内 reward 的 z-score
无需 Value Model: 用组内排名代替
KL: 逐 token 计算
核心区别：
PPO: Advantage = R - V(s)，需要训练一个 Value Network
DPO: 把 RL 问题转化成分类问题——chosen vs rejected 的二元对比
GRPO: Advantage = 组内 z-score，每个 prompt 采样多个响应互相比较

显存占用对比 (以 7B 模型为例, bf16):
PPO: Policy(14GB) + Ref(14GB) + RM(14GB) + Value(14GB) ≈ 56GB (不含优化器状态)
DPO: Policy(14GB) + Ref(14GB) ≈ 28GB
GRPO: Policy(14GB) + Ref(14GB) + RM(14GB) ≈ 42GB (但省了 Value 的 14GB)
特性 PPO DPO GRPO
Reward Model 需要 不需要 需要
Value Function 需要 不需要 不需要
显存中模型数 4 2 3
在线/离线 在线采样 离线偏好对 在线采样 G 个
Advantage R - V(s) 隐式 (chosen - rejected) 组内 z-score
KL 约束 序列级 penalty 隐式 (β 控制) 逐 token penalty
训练数据 prompt (在线生成) (chosen, rejected) 对 prompt (在线生成 G 个)
实现复杂度 高 低 中
代表模型 ChatGPT, Claude Llama 2-Chat, Zephyr DeepSeek-R1
年份 2017 2023 2024
三者共识:
1. 都需要 Reference Model 作为锚点，防止偏离太远
2. 都有 KL 约束（显式或隐式），控制探索幅度
3. 本质都是 相对比较：好的回答 vs 差的回答
4. 最终目标一致：π(y_good) > π(y_bad)
